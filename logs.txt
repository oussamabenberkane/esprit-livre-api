how do I restore it, just tell me how dont do it
Restore is the reverse of what the backup script did. Extract the archive set from ~/backups/el/latest/ and feed each piece back to its service.

1. Stage the files on the prod host

rsync -a ~/backups/el/latest/ personal:/root/.restore-staging/
ssh personal
cd /root/.restore-staging
2. Postgres (postgres.dump)

# Stop the app so nothing writes during restore
docker compose -f /root/esprit-livre/docker-compose.yml stop espritlivre-app

# Drop & recreate the DB (DESTRUCTIVE)
docker exec -i espritlivre-postgres dropdb   -U el --if-exists el-prod-db
docker exec -i espritlivre-postgres createdb -U el el-prod-db

# Restore
docker cp postgres.dump espritlivre-postgres:/tmp/postgres.dump
docker exec espritlivre-postgres pg_restore -U el -d el-prod-db --clean --if-exists /tmp/postgres.dump
docker exec espritlivre-postgres rm /tmp/postgres.dump
3. Keycloak realm (keycloak-realm.tar.gz)
Easiest path is import-on-start (the realm must not already exist, or use --override true):


tar -xzf keycloak-realm.tar.gz                      # yields keycloak-export/jhipster-realm.json
docker cp keycloak-export espritlivre-keycloak:/tmp/kc-import
docker compose -f /root/esprit-livre/docker-compose.yml stop espritlivre-keycloak
# Run a one-shot import (same port conflict caveat — run with the main KC stopped):
docker run --rm \
  --network esprit-livre_default \
  -v kc-import-vol:/tmp/kc-import \
  quay.io/keycloak/keycloak:26.2.3 \
  /opt/keycloak/bin/kc.sh import --dir /tmp/kc-import --override true
docker compose -f /root/esprit-livre/docker-compose.yml start espritlivre-keycloak
Simpler alternative if you just need the realm back on a fresh KC: drop jhipster-realm.json into src/main/docker/realm-config/ before first start.

4. Media volume (media.tar.gz)

docker run --rm \
  -v espritlivre_api_media:/dst \
  -v /root/.restore-staging:/src:ro \
  alpine sh -c 'cd /dst && tar -xzf /src/media.tar.gz'
(If you want a clean replace, add rm -rf /dst/* before the tar -xzf.)

5. Host config (host-config.tar.gz)
Contains /root/esprit-livre (compose, nginx, .env). Only extract if you're rebuilding the host:


tar -C /root -xzf host-config.tar.gz     # recreates /root/esprit-livre
6. TLS certs (tls-letsencrypt.tar.gz)

tar -C /etc -xzf tls-letsencrypt.tar.gz  # recreates /etc/letsencrypt
systemctl reload nginx                   # or restart the nginx container
7. Bring everything back up

docker compose -f /root/esprit-livre/docker-compose.yml up -d
rm -rf /root/.restore-staging
Sanity check before you start: sha256sum -c against MANIFEST.txt to confirm the archives aren't corrupt. Order matters: Postgres + Keycloak first, then media, then start the app — the app will crash-loop if it boots before its DB/realm are in place.
